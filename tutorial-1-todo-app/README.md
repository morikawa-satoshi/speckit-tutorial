# Tutorial 1: 基礎編 - ToDoアプリ

## 目標
spec-kitの基本的なワークフローを理解し、シンプルなToDoリストアプリケーションを仕様から実装まで完成させます。

## 所要時間
約30分

## 学習内容
- spec-kitの基本コマンド
- 仕様駆動開発の基本フロー
- Constitution → Specify → Plan → Tasks → Implement のサイクル

## Step 1: プロジェクト初期化

まず、spec-kitプロジェクトを初期化します。

```bash
cd tutorial-1-todo-app
specify init todo-app --ai claude
```

このコマンドは `.speckit/` ディレクトリを作成し、spec-kitのスラッシュコマンドが使えるようになります。

## Step 2: Constitution (憲法)の作成

プロジェクトの原則と制約を定義します。Claude CodeまたはCursorなどのAIエージェントで以下のコマンドを実行してください。

```
/speckit.constitution
```

以下の内容を含めるように指示してください:

**指示例**:
```
ToDoアプリの開発原則を定義してください:
- シンプルで直感的なUI
- データはローカルストレージに保存
- バニラJavaScript (フレームワーク不使用)
- レスポンシブデザイン
- アクセシビリティを考慮
```

生成されるファイル: `.speckit/constitution.md`

### 確認ポイント
- [ ] `.speckit/constitution.md` が作成されている
- [ ] プロジェクトの原則が明確に記載されている
- [ ] 技術的制約が定義されている

## Step 3: Specification (仕様)の作成

機能要件を定義します。

```
/speckit.specify
```

**指示例**:
```
ToDoリストアプリの機能仕様を定義してください:

必須機能:
1. ToDoアイテムの追加
2. ToDoアイテムの完了/未完了の切り替え
3. ToDoアイテムの削除
4. ToDoリストの表示
5. 完了済みアイテムのフィルタリング

ユーザーストーリー:
- ユーザーとして、新しいタスクを追加できる
- ユーザーとして、タスクを完了済みにマークできる
- ユーザーとして、不要なタスクを削除できる
- ユーザーとして、すべて/未完了/完了済みのタスクを表示できる
```

生成されるファイル: `.speckit/specifications/todo-app.md`

### 確認ポイント
- [ ] 機能要件が明確に定義されている
- [ ] ユーザーストーリーが含まれている
- [ ] 受け入れ基準が記載されている

## Step 4: Plan (計画)の作成

技術的な実装計画を立てます。

```
/speckit.plan
```

**指示例**:
```
ToDoアプリの技術実装計画を作成してください:

考慮事項:
- HTMLファイル構造
- CSSでのスタイリング方針
- JavaScriptでのデータ管理
- LocalStorageの使用方法
- イベントハンドリング
```

生成されるファイル: `.speckit/plans/todo-app-implementation.md`

### 確認ポイント
- [ ] ファイル構造が定義されている
- [ ] データモデルが明確
- [ ] 実装アプローチが記載されている

## Step 5: Tasks (タスク)の生成

実装可能な単位にタスクを分解します。

```
/speckit.tasks
```

生成されるファイル: `.speckit/tasks/todo-app-tasks.md`

### 期待されるタスク例
1. HTML構造の作成
2. CSSスタイルの実装
3. ToDoアイテムのデータモデル作成
4. LocalStorage操作の実装
5. アイテム追加機能の実装
6. アイテム削除機能の実装
7. 完了状態の切り替え実装
8. フィルタリング機能の実装

### 確認ポイント
- [ ] タスクが適切な粒度に分解されている
- [ ] タスクに優先順位がついている
- [ ] 各タスクが実装可能な単位になっている

## Step 6: Implement (実装)

タスクを1つずつ実装していきます。

```
/speckit.implement
```

**指示例**:
```
todo-app-tasks.mdのタスクを順番に実装してください
```

生成されるファイル:
- `index.html` - メインHTMLファイル
- `styles.css` - スタイルシート
- `app.js` - アプリケーションロジック

### 確認ポイント
- [ ] すべてのファイルが生成されている
- [ ] 仕様で定義した機能が実装されている
- [ ] コードがconstitutionの原則に従っている

## Step 7: 動作確認

ブラウザでアプリケーションを開いて動作確認します。

```bash
# シンプルなHTTPサーバーを起動
python3 -m http.server 8000
# または
npx serve .
```

ブラウザで `http://localhost:8000` を開きます。

### 動作確認チェックリスト
- [ ] ToDoアイテムを追加できる
- [ ] アイテムを完了済みにできる
- [ ] アイテムを削除できる
- [ ] フィルタリングが動作する
- [ ] ページをリロードしてもデータが保持される

## Step 8: Clarify (明確化) - オプション

仕様が不明確な部分があれば明確化します。

```
/speckit.clarify
```

**使用例**:
```
以下の点を明確化してください:
- ToDoアイテムの最大文字数は?
- 編集機能は必要か?
- 優先順位の設定は必要か?
```

## 学んだこと

このチュートリアルで以下を学びました:

1. **Constitution**: プロジェクトの原則と制約の定義
2. **Specify**: 機能要件とユーザーストーリーの作成
3. **Plan**: 技術的な実装計画の立案
4. **Tasks**: 実装可能な単位へのタスク分解
5. **Implement**: 仕様に基づいた実装
6. **Clarify**: 仕様の明確化(オプション)

## トラブルシューティング

### spec-kitコマンドが見つからない
```bash
# spec-kitの再インストール
uv tool install specify-cli --from git+https://github.com/github/spec-kit.git
```

### スラッシュコマンドが動作しない
- Claude CodeまたはCursorなどのAIエージェント内で実行してください
- プロジェクトディレクトリに `.speckit/` フォルダがあることを確認してください

### 生成されたコードが期待と異なる
- `/speckit.clarify` を使って仕様を明確化してください
- constitutionやspecificationを見直して、より詳細に記述してください
